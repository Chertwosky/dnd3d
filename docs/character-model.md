# Модель Character

Модель `Character` описывает персонажа игрока в кампании и хранит две версии данных импорта:

- `rawJson` — неизменённый исходный JSON-файл, который пользователь импортировал в приложение;
- `normalizedData` — очищенная и приведённая к рабочему формату структура, которую использует интерфейс приложения.

Такой подход позволяет повторно нормализовать персонажа при изменении правил импорта, не теряя оригинальный файл пользователя.

## Поля модели

| Поле | Тип | Назначение |
| --- | --- | --- |
| `id` | `String` UUID | Уникальный идентификатор персонажа. |
| `userId` | `String` UUID | Владелец персонажа. Ссылается на `User.id`. |
| `campaignId` | `String` UUID, nullable | Кампания, к которой привязан персонаж. Ссылается на `Campaign.id`. |
| `name` | `String` | Имя персонажа. |
| `edition` | `String` | Редакция или источник правил, например `dnd-5e`. |
| `level` | `Int` | Уровень персонажа. По умолчанию `1`. |
| `className` | `String`, nullable | Основной класс персонажа. |
| `subclassName` | `String`, nullable | Подкласс или архетип персонажа. |
| `raceName` | `String`, nullable | Раса или происхождение персонажа. |
| `background` | `String`, nullable | Предыстория персонажа. |
| `alignment` | `String`, nullable | Мировоззрение персонажа. |
| `experience` | `Int` | Количество опыта. По умолчанию `0`. |
| `stats` | `Json` object | Характеристики персонажа: сила, ловкость, телосложение, интеллект, мудрость, харизма и производные значения. |
| `skills` | `Json` object | Навыки, владения навыками, бонусы и итоговые значения проверок. |
| `saves` | `Json` object | Спасброски, владения спасбросками и итоговые модификаторы. |
| `vitality` | `Json` object | Рабочий блок здоровья и защиты: хиты, временные хиты, кость хитов, класс брони, инициатива, скорость и состояния. |
| `spells` | `Json` array | Список заклинаний, ячеек, подготовленных заклинаний и параметров заклинателя. |
| `items` | `Json` array | Инвентарь, экипировка, оружие, броня, расходники и деньги. |
| `notes` | `Json` object | Заметки игрока, черты, особенности, связи, цели или произвольные текстовые блоки. |
| `avatar` | `String`, nullable | Ссылка, идентификатор ассета или внешний URL аватара персонажа. |
| `rawJson` | `Json` | Исходный импортированный файл без очистки и преобразований. |
| `normalizedData` | `Json` object | Нормализованная рабочая структура приложения после валидации и очистки импорта. |

## Правила хранения импортированных данных

1. При импорте JSON-файла приложение сохраняет полный исходный документ в `rawJson` без удаления неизвестных полей.
2. Импортёр валидирует обязательные данные и формирует `normalizedData` в стабильной структуре приложения.
3. UI и игровые механики должны читать рабочие значения из типизированных верхнеуровневых полей и `normalizedData`, а не из `rawJson`.
4. `rawJson` используется для аудита, повторной обработки импорта и отладки несовместимых файлов.
5. Если импортированный файл содержит дополнительные блоки, которые пока не поддержаны интерфейсом, их можно оставить только в `rawJson` до появления соответствующей нормализации.

## Соответствие Prisma

В Prisma модель использует snake_case-имена колонок для базы данных и camelCase-поля в приложении:

- `userId` хранится в колонке `user_id`;
- `campaignId` хранится в колонке `campaign_id`;
- `className` хранится в колонке `class_name`;
- `subclassName` хранится в колонке `subclass_name`;
- `raceName` хранится в колонке `race_name`;
- `rawJson` хранится в колонке `raw_json`;
- `normalizedData` хранится в колонке `normalized_data`.

`Character` связан с `User`, опционально связан с `Campaign`, может быть выбран участником кампании через `CampaignMember` и может использоваться токенами карты через `Token`.
